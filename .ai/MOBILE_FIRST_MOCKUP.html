  • ‚úÖ 120px images: Deployed! LEFT side, content on RIGHT (using free real estate!)
  • ‚úÖ No SKU/UPC labels: Pill design + monospace font = self-identifying system data
  • ‚úÖ Price proximity: Retail price NEXT TO $0.01 for instant value comparison
  • ‚úÖ Removed "Has photo" filter: ~25% more screen space for cards
  • ‚úÖ Recency filter: 24h/3d/7d/30d (user-focused, not time-box focused)
  • ‚úÖ "Hunter's Intel": SINGLE dismissible banner (not per-card!)
  • ‚úÖ Clickable UPC pill: Shows barcode modal, no "UPC:" label clutter
  • ‚úÖ Icon-only Save: High-contrast, less button weight, focus stays on data
  • üì± Efficiency: ~3+ cards per screen (vs 1.5 before)

üìä Key Design Decisions

1. Mobile-First Approach ‚úÖ

Prioritized mobile layout improvements because "battle is won or lost on mobile." Removing the "has photo" filter alone frees significant vertical space on small screens.

Horizontal Layout (Free Real Estate!) ‚úÖ

Changed to horizontal: image LEFT (120px), content RIGHT. Uses the empty space next to the image instead of stacking everything vertically. More efficient on mobile screens Kept the current vertical card structure (image top, content below) instead of transforming to horizontal. This is what users are familiar with and works well on mobile.

3. Image Size Already Deployed ‚úÖ

The 120px image size experiment has already been deployed and tested. This mockup reflects production reality.

4. "Pro Tool" Data Efficiency

No labels: SKU/UPC displayed as pills with monospace font (self-identifying).
Price proximity: Retail price directly next to $0.01 for instant value comparison.
Button weight: Icon-only Save button keeps visual focus on the data, not chrome.

5. "Hunter's Intel" Branding (One Banner)

Replaces warning-toned "Please Read" with ONE dismissible "Hunter's Intel" banner above the card list (not per-card). Blue, informational tone. Builds community trust without cluttering every card.
