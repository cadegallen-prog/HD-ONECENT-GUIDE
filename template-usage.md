## Quick Start

### Creating a New Page (5-Minute Version)

1. **Copy an existing page** as a starting point:
   ```bash
   cp what-are-pennies.html new-page.html
   ```

2. **Update the `<head>` section**:
   - Change `<title>`
   - Update meta description
   - Update Open Graph tags
   - Update structured data

3. **Replace the `<h1>` and content** in `<main>`

4. **Add to navigation** in ALL existing HTML files:
   ```html
   <li><a href="new-page.html">New Page</a></li>
   ```

5. **Add to PAGE_ORDER** in `scripts.js`:
   ```javascript
   const PAGE_ORDER = [
     'index.html',
     // ... existing pages ...
     'new-page.html', // Add here
     'about.html'
   ];
   ```

6. **Test** the page in a browser

## Page Creation Workflow

### Step 1: Plan Your Content

Before coding, answer these questions:

- **Purpose**: What is this page's goal?
- **Audience**: Beginner, intermediate, or advanced?
- **Length**: How much content? (Estimate read time)
- **Type**: Guide, reference, explanation, or list?
- **Related Pages**: What pages should link here?

### Step 2: Choose a Base Template

Start with a page that's similar to what you're creating:

| Page Type | Base Template | Reason |
|-----------|---------------|--------|
| Explanatory article | `what-are-pennies.html` | Good structure for teaching concepts |
| Step-by-step guide | `digital-prehunt.html` | Has procedure steps and checklists |
| Reference tables | `clearance-lifecycle.html` | Complex tables and quick reference cards |
| FAQ/Questions | `faq.html` | Accordion structure |
| List/Resources | `resources.html` | Card layout and external links |

### Step 3: Set Up the File

Copy the chosen template:

```bash
cp what-are-pennies.html my-new-page.html
```

### Step 4: Update Metadata

Edit the `<head>` section:

#### Required Updates

1. **Page Title**
   ```html
   <title>My Page Title - Home Depot Penny Items Guide</title>
   ```

2. **Meta Description** (155 characters max)
   ```html
   <meta name="description" content="Brief, compelling description for search engines">
   ```

3. **Keywords**
   ```html
   <meta name="keywords" content="keyword1, keyword2, keyword3">
   ```

4. **Canonical URL**
   ```html
   <link rel="canonical" href="https://yourdomain.com/my-new-page.html">
   ```

5. **Open Graph Tags**
   ```html
   <meta property="og:title" content="My Page Title">
   <meta property="og:description" content="Description for social media">
   <meta property="og:url" content="https://yourdomain.com/my-new-page.html">
   ```

6. **Twitter Card**
   ```html
   <meta name="twitter:title" content="My Page Title">
   <meta name="twitter:description" content="Description for Twitter">
   <meta name="twitter:url" content="https://yourdomain.com/my-new-page.html">
   ```

7. **Structured Data**
   ```html
   <script type="application/ld+json">
   {
     "@context": "https://schema.org",
     "@type": "Article",
     "headline": "My Page Title",
     "description": "Page description"
   }
   </script>
   ```

### Step 5: Build the Content

Replace the content in `<main>`:

#### Required Content Structure

```html
<main>
  <!-- 1. Breadcrumbs (auto-generated by JS) -->
  <div class="breadcrumbs"></div>

  <!-- 2. Page Title -->
  <h1>My Page Title</h1>

  <!-- 3. Table of Contents (auto-generated by JS) -->
  <div class="table-of-contents">
    <h3>On This Page</h3>
  </div>

  <!-- 4. Page Metadata -->
  <div class="page-metadata">
    <span class="meta-badge icon-time difficulty-beginner">5 min read</span>
    <span class="meta-badge icon-level difficulty-beginner">Beginner</span>
  </div>

  <!-- 5. Summary Box -->
  <div class="summary-box">
    <h2 class="icon-star">Page Summary</h2>
    <p>Brief overview of what this page covers.</p>
  </div>

  <!-- 6. Main Content -->
  <!-- Your content here -->

  <!-- 7. Related Topics (optional but recommended) -->
  <div class="related-topics-section">
    <h3 class="icon-link">Related Topics</h3>
    <ul>
      <li><a href="page1.html">Page 1</a> ‚Äî Description</li>
      <li><a href="page2.html">Page 2</a> ‚Äî Description</li>
    </ul>
  </div>

  <!-- 8. Page Navigation (auto-populated by JS) -->
  <div class="page-nav-buttons">
    <a href="#" class="nav-prev-btn"></a>
    <a href="#" class="nav-next-btn"></a>
  </div>
</main>
```

### Step 6: Add Navigation Links

**CRITICAL**: Update navigation in ALL HTML files:

1. Open each existing `.html` file
2. Find the `<nav>` section
3. Add your new page in the appropriate location:
   ```html
   <li><a href="my-new-page.html">My New Page</a></li>
   ```

**Tip**: Use search and replace in your editor to update all files at once.

### Step 7: Update JavaScript Configuration

Edit `scripts.js`:

1. **Add to PAGE_ORDER** (determines prev/next buttons):
   ```javascript
   const PAGE_ORDER = [
     'index.html',
     'quick-start.html',
     // ... existing pages ...
     'my-new-page.html', // Add in logical reading order
     'about.html'
   ];
   ```

2. **Add to PAGE_TITLES**:
   ```javascript
   const PAGE_TITLES = {
     // ... existing pages ...
     'my-new-page.html': 'My New Page'
   };
   ```

### Step 8: Test Your Page

1. **Open in browser**: `file:///path/to/my-new-page.html`
2. **Check navigation**: All links work?
3. **Test responsiveness**: Resize browser window
4. **Verify JavaScript**:
   - Breadcrumbs appear?
   - Table of contents generated?
   - Prev/Next buttons work?
   - Back to top button appears on scroll?
5. **Test accessibility**:
   - Tab through interactive elements
   - Check color contrast
   - Verify heading hierarchy

## JavaScript Features

The template includes automatic JavaScript features:

### Auto-Generated Elements

| Feature | Element | Generated From |
|---------|---------|----------------|
| Breadcrumbs | `<div class="breadcrumbs">` | Current page location |
| Table of Contents | `<div class="table-of-contents">` | H2 and H3 headings |
| Previous/Next Buttons | `.nav-prev-btn`, `.nav-next-btn` | PAGE_ORDER array |
| Active Nav Link | `.nav-links a.active` | Current page URL |
| Back to Top | `.back-to-top` | Scroll position |

### Progress Tracking

Pages are automatically marked as "read" when visited. Progress is stored in `localStorage`.

**Reset Progress**:
```javascript
ProgressTracker.resetProgress();
```

### Theme Toggle

Users can switch between light and dark themes. Preference is saved in `localStorage`.

### Font Size Controls

Three sizes available: small (16px), medium (18px), large (20px). Saved in `localStorage`.

### Search Functionality

Site-wide search is available via:
- Click search bar
- Press `Ctrl+K`
- Press `/` key

Search index is auto-generated from page content.

## Troubleshooting

### Problem: Navigation doesn't highlight active page

**Solution**: Check that the filename in PAGE_ORDER matches exactly:
```javascript
// Correct
'my-page.html'

// Wrong
'my_page.html'
'MyPage.html'
```

### Problem: Table of contents doesn't generate

**Solution**: Ensure H2/H3 headings are in `<main>`, not in `<nav>` or `<footer>`.

### Problem: Previous/Next buttons don't work

**Solution**: Add page to PAGE_ORDER array in scripts.js in the correct position.

### Problem: Breadcrumbs don't appear

**Solution**: Ensure `<div class="breadcrumbs"></div>` is inside `<main>` and scripts.js is loaded.

### Problem: Search doesn't find my page

**Solution**: Search indexes pages on load. Clear browser cache and reload.

### Problem: Mobile layout broken

**Solution**:
1. Check that `<meta name="viewport">` is present
2. Wrap wide tables in `<div class="table-enhanced">`
3. Test with browser dev tools mobile emulation

### Problem: Dark theme doesn't apply

**Solution**:
1. Ensure CSS has dark theme variables defined
2. Check that JavaScript is loading correctly
3. Try toggling theme and checking localStorage

## Best Practices

### DO ‚úÖ

- Start with an existing page as a template
- Keep heading hierarchy logical (H1 ‚Üí H2 ‚Üí H3)
- Use semantic HTML (`<nav>`, `<main>`, `<footer>`)
- Wrap tables in `.table-enhanced` div
- Add ARIA labels to interactive elements
- Test on mobile devices
- Use callouts for important information
- Link to related pages
- Keep page length reasonable (5-10 min read)
- Update navigation in ALL pages

### DON'T ‚ùå

- Create multiple H1 headings on one page
- Use inline styles (use CSS classes)
- Forget to update PAGE_ORDER in scripts.js
- Skip accessibility testing
- Use "click here" link text
- Forget to add breadcrumbs div
- Mix tab and space indentation
- Hardcode previous/next links
- Skip the summary box
- Add new pages without updating navigation everywhere
